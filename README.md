# EcoCommerce AI Platform

## Project Overview

EcoCommerce AI Platform is an **AI-powered e-commerce automation system** designed to simplify product catalog management and improve customer support for sustainable online stores.

The platform demonstrates how **AI prompt engineering and automation** can streamline common e-commerce tasks such as product categorization, SEO tag generation, and customer support.

The system includes two major AI-driven modules:

1. **AI Auto-Category & Tag Generator** – Automatically categorizes products and generates SEO tags.
2. **AI WhatsApp Support Bot** – Provides automated responses to customer queries and escalates complex issues.

The application follows a **client–server architecture** consisting of a React frontend, a Node.js backend, AI services, and a MongoDB database.

The focus of the project is not only functionality but also **clear architecture, structured AI prompts, and maintainable system design**.

# Backend Setup

```bash
cd server
npm install
```

Create a `.env` file in the `server` directory with the following variables:

```
PORT=8000
MONGODB_URI=your_mongodb_connection_string
CORS_ORIGIN=frontend_url
JWT_SECRET=your_jwt_secret
GOOGLE_CLIENT_SECRET=your_client_secret
GOOGLE_CLIENT_ID=your_id
GOOGLE_REDIRECT_URI=URL
EMAIL_USER=your_mail
EMAIL_Pass=your_pass
AI_API_KEY=your_ai_api_key
WHATSAPP_API_KEY=your_whatsapp_api_key

```

Start the server:

```
npm run dev
```

# System Architecture Overview

The system follows a **layered architecture** where each component has a clearly defined responsibility.

User / Admin
      │
      ▼
Frontend (React Dashboard)
      │
      │ REST API
      ▼
Backend (Node.js / Express)
      │
      ├── AI Auto-Category & Tag Generator
      │
      ├── AI WhatsApp Support Bot
      │
      ▼
AI Model / LLM API
      │
      ▼
MongoDB Database


## Architecture Explanation

### Frontend Layer

The frontend provides the user interface for administrators to manage the product catalog and interact with the AI-powered features.

Responsibilities include:

* Product creation interface
* Display of AI-generated product metadata
* Admin dashboard
* API communication with backend services

---

### Backend Layer

The backend acts as the **core application layer** responsible for:

* Processing API requests
* Generating AI prompts
* Handling AI responses
* Managing products and conversations
* Storing and retrieving database records

---

### AI Layer

The AI layer processes prompts generated by the backend and returns structured results used by the platform.

AI is responsible for:

* Product categorization
* SEO tag generation
* Sustainability attribute detection
* Customer support responses

---

### Database Layer

MongoDB stores structured application data including:

* Product metadata
* AI-generated tags and filters
* Customer conversations
* Order information

MongoDB’s document structure works well with **AI-generated JSON outputs**.

# Module 1: AI Auto-Category & Tag Generator

## Purpose

E-commerce platforms require accurate product categorization and SEO optimization.
This module uses AI to automatically analyze product details and generate structured metadata.

---

## Features

### Auto-Assign Primary Category

The AI determines the most appropriate category from a predefined list such as:

* Clothing
* Kitchen
* Personal Care
* Home & Living
* Accessories
* Eco Lifestyle

---

### Suggest Sub-Category

The system suggests a more specific classification within the main category.

Example:

Primary Category: Kitchen
Sub-Category: Reusable Drinkware

---

### Generate SEO Tags

The system generates **5–10 SEO optimized tags** to improve search engine visibility.

Example tags:

* eco water bottle
* bamboo bottle
* reusable bottle
* plastic free bottle
* sustainable drinkware

---

### Sustainability Filters

The AI identifies eco-friendly attributes and suggests sustainability filters:

* Plastic-Free
* Compostable
* Vegan
* Recycled
* Biodegradable

---

### Structured JSON Output

The AI returns results in structured JSON format.

Example:

```json
{
  "primaryCategory": "Kitchen",
  "subCategory": "Reusable Drinkware",
  "seoTags": [
    "eco water bottle",
    "bamboo bottle",
    "plastic free bottle",
    "reusable bottle"
  ],
  "sustainabilityFilters": [
    "plastic-free",
    "reusable",
    "eco-friendly"
  ]
}
```

This output is validated and stored in the database.

# Module 2: AI WhatsApp Support Bot

## Purpose

The AI WhatsApp bot provides automated customer support by responding to common e-commerce queries.

---

## Features

### Order Status Queries

Customers can ask:

* “Where is my order?”
* “What is my order status?”

The system retrieves order data and generates a response.

Example:

```
Your order #1245 has been shipped and is expected to arrive in 2 days.
```

---

### Return Policy Assistance

The bot answers common policy questions.

Example queries:

* “What is your return policy?”
* “Can I return a product?”

Example response:

```
Our return policy allows returns within 30 days of delivery if the product is unused and in its original packaging.
```

---

### Escalation of High-Priority Issues

The AI detects complex issues such as:

* Refund requests
* Payment disputes
* Product complaints

These queries are escalated to human support.

Example escalation:

```
Your request has been forwarded to our support team. A representative will contact you shortly.
```

---

### Conversation Logging

All interactions are stored for monitoring and analytics.

Stored information includes:

* Customer message
* AI response
* Timestamp
* Escalation status

# AI Prompt Design

AI prompt design is a critical part of the system.
Prompts are structured to ensure **consistent and predictable responses from the AI model**.

---

## Prompt Design Goals

The prompts are designed to:

* Produce structured JSON outputs
* Improve product classification accuracy
* Generate SEO-friendly tags
* Detect sustainability attributes
* Provide clear customer support responses

---

## Prompt Example – Category & Tag Generator

```
You are an expert eco-commerce product analyst.

Analyze the following product details and generate structured metadata.

Product Information:
- Product Name: {product_name}
- Description: {description}
- Material: {material}

Tasks:
1. Assign the most appropriate primary category.
2. Suggest a sub-category.
3. Generate 5–10 SEO tags.
4. Suggest sustainability filters such as plastic-free, compostable, vegan, or recycled.

Return the output in JSON format with:
primaryCategory
subCategory
seoTags
sustainabilityFilters
```

---

## Prompt Example – WhatsApp Support Bot

```
You are an AI customer support assistant for an eco-commerce store.

Guidelines:
1. If the user asks about order status, retrieve order data from the database.
2. If the user asks about return policy, provide the official policy.
3. If the query involves refunds or complaints, escalate to human support.

Respond clearly and professionally.
```

# Assumptions & Design Decisions

* REST API architecture was chosen for simplicity and maintainability.
* MongoDB Atlas was selected due to flexible document storage.
* AI responses are structured in JSON to simplify backend processing.
* Backend handles AI prompt construction to keep frontend logic simple.
* Conversations are logged to enable analytics and AI improvement.
